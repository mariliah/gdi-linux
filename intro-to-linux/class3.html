CIA Triad

What is the CIA Triad?

In Cybersecurity, CIA stands for: Confidentiality, Integrity, and Availability. These principles form the foundation of information security practices and guide efforts to protect sensitive data and systems from various threats.

Confidentiality

In Linux, user and permission management helps enforce condifentiality by controlling access to sensitive files, directories, and resources. By assigning appropriate permissions, system administrators can ensure that only authorized users have access to confidential information. For example, sensitive configuration files containing passwords or cryptographic keys can be restricted to read-only access by the owner, preventing unauthorized users from viewing or modifying them.

Integrity

User and permission management also contributes to maintaining the integrity of data and system files in Linux. By carefully managing user privileges and permissions, administrators can prevent unauthorized users from modifying critical system files or data. For instance, restricting write access to system binaries and configuration files helps prevent malicious users from tampering with them, ensuring the integrity and reliability of the system.

Availability

Effective user and permission management in Linux ensures the availability of system resources and data by preventing unauthorized users from disrupting or interfering with system operations. By controlling access to critical resources and directories, administrators can mitigate the risk of denial-of-service attacks and unauthorized modifications that could impact system availability. Additionally, implementing proper user authentication mechanisms helps ensure that system resources are accessible only to authorized users, maintaining uninterrupted service availability.

What is a 'user' in Linux?

Under Linux, every file and program must be owned by an entity (or user). This can be a human, a daemon, a thing, a group, and so on. Each user has a unique identifier called user ID (UID). Each user must belong to at least one group -- a collection of users. Users may belong to more than one group. Groups also have unique identifiers, called group IDs (GIDs).
