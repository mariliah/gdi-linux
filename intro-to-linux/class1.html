Background of Linux

  • Based on UNIX operating system created by AT&T in 1971.
  • Created by Linus Torvalds in 1991, who was a student at University of Helsinki, Finland at the time. He also created the distributed version control system Git.
  • Open source code.
  • Which other operating system is based on UNIX, but is NOT open source?
  • UNIX was a registered trademark owned by AT&T, which was eventually made into a commercial product.
  • Written in the C programming language which made UNIX a portable operating system that could be installed in any computer, and improve development environments.
  • Linus Torvalds, a university student in 1991 wrote a message on comp.os.minix stating he was working on an operating system, as a hobby, based of Minix which was a copy of UNIX operating system itself.

Back up! What is a distribution?

Recall the "kernel." It could be seen as a seed, or also as the foundation of an operating system. A kernel is the core software that manages hardware resources and allows other programs to communicate.

A distribution (distro for short) is a complete house built on this foundation. It includes the kernel, but also adds essential tools and functionalities.

Tools and Functionalities

Package management system: Imagine a toolbox. A distro's package manager lets you easily install, update, and remove software applications.

Shell and utilities: The shell is a command-line interface that allows you to interact with the system. Utilities are like individual tools that perform specific tasks.

Desktop environment(optional): This provides the graphical user interface (GUI) that everybody is familiar with on Windows or macOS. Some distros cater to server use and may not include a GUI by default!

Lastly, some distros come pre-loaded with a variety of applications for common tasks like browsing the web, writing documents, or playing media.

Command Line Interface

The command line interface (CLI), also known as a terminal emulator (or simply terminal) is a text-based command interpreter.

Command Syntax

                    
                        $ command -options targets

pwd

(Print Working Directory)

Type this command whenever you want to see what directory (folder) you are working in.

What is a directory?

A directory, also refered to as "folder" are containers for files or other directories.

File Paths

A sequence of nested directories, with a file or directory at the end.

Each directory or file is separated by a forward slash "/" (unlike Windows which is "\").

Useful shortcuts

  • Current directory: .
  • Parent directory: ..
  • Home directory: ~
  • Root directory: /

cd

Change Directory

Use the cd command to change directories. This command expects a file path as an arguement.

If no file path is given, it assumes your home directory (~) by default.

ls and ls -la

List Command

Use the ls command to view content of a directory.

mkdir

Make Directory Command

Use the mkdir command to create a directory.

rmdir or rm

Remove Directory Command

Use the rmdir command to remove a directory.

WARNING ABOUT RMDIR

When you remove a directory, or a file, it does not go into "Trash", it is permanently deleted! Please do not ever run the command rm -rf /*, because this will delete all the files and directories of your computer!
