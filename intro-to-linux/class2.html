Filesystem Hierarchy Standard

Absolute vs Relative Paths

Absolute Path: An absolute path specifies the location of a file or directory from the root directory (/) of the file system. It starts from the root directory and includes all directories down to the specified file or directory. For example, '/home/user/documents/file.txt' is an absolute path where / represents the root directory, home is a directory within the root directory, user is a directory within home, and so on until file.txt.

Relative Path: A relative path specifies the location of a file or directory relative to the current working directory. It does not start from the root directory but rather from the current directory. For example, if the current working directory is '/home/user/', then a relative path like 'documents/file.txt' would refer to the 'file.txt' within the documents directory located in the current directory.

What is a directory?

A directory, also refered to as "folder" are containers for files or other directories.

File Paths

A sequence of nested directories, with a file or directory at the end.

Each directory or file is separated by a forward slash "/" (unlike Windows which is "\").

Useful shortcuts

  • Current directory: .
  • Parent directory: ..
  • Home directory: ~
  • Root directory: /

cd

Change Directory

Use the cd command to change directories. This command expects a file path as an arguement.

If no file path is given, it assumes your home directory (~) by default.

ls and ls -la

List Command

Use the ls command to view content of a directory.

mkdir

Make Directory Command

Use the mkdir command to create a directory.

rmdir or rm

Remove Directory Command

Use the rmdir command to remove a directory.

WARNING ABOUT RMDIR

When you remove a directory, or a file, it does not go into "Trash", it is permanently deleted! Please do not ever run the command rm -rf /*, because this will delete all the files and directories of your computer!
